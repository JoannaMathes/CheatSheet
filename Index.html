Semantic HTML Cheatsheet

The day of using <div> for everything is in the past!

Let's get specific with semantics!

Semantic HTML automatically tells us more about the content within.
This improves SEO and screen reader accessibility among many other benefits

Syntax Explanation
<header>
</header>		 This is a better alternative to having the top section be a div named header
<nav>
</nav>		 This tag is only for the navigation and is much more semantic than having a div at the top named navigation
<main>
</main>		 This is best used to nest the main elements of the page between the header and nav at the top and the secondary sections and footer at the bottom
<section>
</section>		 This tag is a better way of dividing up the page into main sections instead of using a div and giving a class correlating with the contents of each section
<article>
</article>		 This is best suited to encapsulate articles inside a section
<footer>
</footer>		 Best used for the bottom most section of the web page instead of naming a div footer
